A magnet on the faucet lever passes a sensor wired to the Pi: while the magnet is near, the video pauses; the moment it moves away, it resumes. Vanitas Veritas · Build Guide How the loop is generated
dreamed anchors
+ smooth drift
Re-dreaming every frame boils and flickers; warping pixels reads as water. Both were tried and rejected. The engine instead decouples what the loop looks like from how it moves.
1
Dream a sparse ring of anchors
An img2img pass re-imagines blended source frames into the style. Anchors at the sources stay discernible; anchors between are dreamed harder — gooey, uncanny. Encoded to latents and cached.
2
Interpolate — don’t re-dream
Between anchors the latents are blended along a Catmull-Rom spline and decoded. Decoding is deterministic, so motion is glassy-smooth with zero flicker — and cheap (~1 s/frame), so slowness is free.
3
Grade — don’t warp
A finish pass with no geometric distortion: gloss bloom, deepened blacks, a gold tint in the highlights, film grain, a grime vignette, and a palette lock holding the cool-silver + gold.
Reproduce
python3 morph_v44.py --fresh # omit --fresh to reuse cache
Knobs: D anchor density · M frames/gap (length + slowness) · LOW_STR / HIGH_STR discernible ↔ gooey.

02 / 05Interaction Build Guide
Vanitas Veritas · Build Guide Build it · on your computer + on the Pi
Eight steps, four phases.
Do them in order; type the commands exactly as written.
Phase 1 · On your computer
1
Put the software on the SD card
Open Raspberry Pi Imager (raspberrypi.com/software). Choose Device = Pi 4, OS = Raspberry Pi OS (64-bit), Storage = your card. Edit Settings: set username pi + a password, your Wi-Fi, enable SSH. Write, then eject.
Phase 2 · On the Pi
2
First boot & install the player
Insert the card, connect the screen (micro-HDMI nearest the USB-C corner) + a keyboard, power on. At the login, enter your username + password, then install the player:
sudo apt update && sudo apt install -y mpv python3-gpiozero
3
Put your video on the Pi
Name the loop loop.mp4 (H.264, 4:3, seamless). Copy via USB stick:
cp /media/pi/NAME/loop.mp4 /home/pi/loop.mp4
4
Create the program
Run nano /home/pi/vanitas.py and type this exactly. Save with Ctrl+O, exit Ctrl+X.
import subprocess, socket, json, time from gpiozero import Button subprocess.Popen(["mpv","--fullscreen", "--loop-file=inf","--hwdec=auto","--vo=gpu", "--no-osc","--input-ipc-server=/tmp/mpvsocket", "/home/pi/loop.mp4"]) time.sleep(4) def cmd(p): s = socket.socket(socket.AF_UNIX) s.connect("/tmp/mpvsocket") s.send((json.dumps({"command": ["set_property","pause",p]})+"\n").encode()) s.close() btn = Button(17, pull_up=True) btn.when_pressed = lambda: cmd(True) btn.when_released = lambda: cmd(False) from signal import pause; pause()
The magnet on GPIO17 pauses on the current frame; releasing it plays. The next page wires it to auto-start and lock down.

Vanitas Veritas · Build Guide Auto-start · wire · lock down
5
Turn it on for good
Create /etc/systemd/system/vanitas.service so it relaunches on boot and after any crash:
[Service] ExecStart=/usr/bin/python3 /home/pi/vanitas.py Restart=always User=pi [Install] WantedBy=multi-user.target
sudo systemctl enable vanitas.service && sudo reboot
Phase 3 · Wire & mount
6
Wire the sensor — Pi powered off
The hall module has three pins. Use 3.3V (pin 1), not 5V.
VCC→ physical pin 1 · 3.3V
GND→ physical pin 6
OUT→ physical pin 11 · GPIO17
7
Aim the magnet
Glue the magnet to the moving part of the lever; stick the sensor to the fixed body, facing its path. Power on, slide the sensor until a normal pull reliably pauses with a little margin, then fix it with epoxy.
Phase 4 · Lock it down
8
Make it unbreakable — do this last
Kill Wi-Fi so there’s nothing to glitch: sudo rfkill block wifi
Read-only card so a yanked plug can’t corrupt it: raspi-config → Performance Options → Overlay File System → Enable → reboot.
Burn in: run it 24–48 h, then pull the plug ~10 times. Every time it must return to the playing loop with the handle still pausing. If it does, it’s gallery-ready.
The only still moment in the room is a held breath.
Pausing is the active command, so if the sensor ever fails the loop simply keeps running — it can never fall to a black screen. Plays by default; the gesture only ever subtracts motion.

Vanitas Veritas · Build Guide If something’s wrong · why it’s built this way
If something’s wrong
Black screenThe micro-HDMI must be in the Pi port nearest the USB-C corner. Reseat the cable; check the screen is powered and on the right input. Some cables are charge-only — try another.
Plays but won’t pauseRe-check the three sensor wires and pin numbers. Run python3 /home/pi/vanitas.py by hand and move the magnet. If nothing, swap in another sensor from the 5-pack.
Pause feels laggyMake sure the video is H.264 sized to the screen (Step 3). A huge or wrong-format file makes the Pi struggle.
Won’t start on bootRun sudo systemctl status vanitas to read the error — usually a typo in the program or service file. Re-open with nano and compare line by line.
Why it’s built this way
Feels instant
A hall sensor (not a clicky switch) has no bounce and nothing to wear out over thousands of pulls. The program reacts the instant the pin changes (~1 ms) and pauses on the current frame — you feel the freeze in the same blink you pull. That immediacy is what makes the piece feel alive instead of like a screensaver.
Can’t break
Plays by default, so a failed sensor just leaves the loop running — never a black screen. The read-only card means power cuts can’t corrupt it, and Restart=always relaunches any crash. No moving parts in the trigger. These three are why it survives three unattended weekends.
